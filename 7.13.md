# KT6 项目阶段进展简报（7.13）

## 一、阶段结论

目前已完成 **KT6 意图驱动 LUI-GUI 联动的可运行工程原型**，不再是单纯的前端动画演示。系统能够由后端 Runtime 根据用户输入选择业务任务链，驱动左侧拓扑定位、分析、高亮、方案确认、动作执行和结果校验，并保存完整任务轨迹。

当前已验证两类场景：

- 用户网速慢：定位用户及关联 AP，分析同频干扰，人工确认后执行射频调优。
- AP 离线：定位故障 AP，分析 PoE 异常，人工确认后执行端口恢复并校验上线。

## 三、实现思路

整体链路为：

```text
用户输入
-> Intent Agent 解析意图与槽位
-> Router 选择业务 Playbook
-> Runtime 编排诊断步骤
-> 页面感知生成统一 Scene Graph
-> 左侧 GUI 根据事件定位和高亮
-> 右侧生成处置方案并等待人工确认
-> Runtime 校验场景版本、方案权限和资源锁
-> 执行动作 Playbook并校验结果
-> 保存任务轨迹与业务记忆
```

职责划分：Agent 负责理解与诊断，Playbook 沉淀可审计的业务任务链，Runtime 负责可靠编排，感知适配器负责把 DOM、Canvas 或 KT5 拓扑统一转换为 Scene Graph，前端只消费事件并呈现联动结果。

Browser Use 可以作为浏览器会话、DOM 获取、截图和通用点击执行能力使用，但其内置视觉不能单独满足拓扑语义重建。对于只能看到像素的封闭 Canvas，当前可选择本地 RapidOCR/OpenCV、HTTP Vision 或 CodeAgent read-tool 三种专用适配器，完成节点、链路和业务 ID 识别；跨帧一致性仍由 KT6 的 Scene revision 与变化检测管理。

## 四、当前边界与下一步

当前业务指标、拓扑数据和设备操作仍使用 Mock Adapter；Canvas 截图是真实采集，但 Demo 节点语义主要来自当前页面暴露的渲染器数据。结构化拓扑文本已可用于语义重建，但被明确标记为非像素、不可执行。生产侧现可通过本地 RapidOCR/OpenCV、HTTP 服务或本机 CodeAgent read 工具读取真实截图，仍需用原图和独立标注验证准确率。

下一阶段重点：

1. 获取同一拓扑的原始 PNG/JPEG 和独立标注，验证真实像素的节点、文字和连线识别。
2. 在生产环境配置 `local_cv_ocr`、`codeagent_cli` 或 HTTP 视觉驱动，用 pixels-only CLI 验证具体 OCR/检测/read/多模态链路。
3. 将 Mock 业务查询和设备下发替换为真实接口，并补齐鉴权、审计和回滚。
4. 联调 KT5 场景化拓扑生成，验证“感知已有拓扑”和“按意图生成拓扑”两种模式统一运行。

**汇报口径：本阶段完成了 KT6 核心架构和端到端 PoC，已经具备接入真实页面、真实业务接口、专用视觉模型及 KT5 的工程基础，但尚未达到生产交付状态。**

## 五、2026-07-14 拓扑界面感知增量

针对“Unicode 拓扑图 + 设备详情表 + 特殊说明”的企业网络样例，已建立严格 Ground Truth 和自动化测试：

- 识别 22 个设备、19 条明确关系；主连通分量 20 个节点，共 3 个连通分量、2 个孤立设备、0 个环。
- `AGG-003` 只在表中出现，因此不根据“三层架构”说明补边；`AP-007` 保留为父节点未知的独立接入设备。
- Trunk 和 6 个“AP群”保留为非设备视觉组。
- `LSW`、`?`、`ZTE`、`FS`、`ONU` 保留原始不确定证据，不升级为未经证实的设备事实。
- 文本、外部 OCR、Renderer 和 CanvasVision 使用不同 provenance；只有实际读取截图像素的适配器可声明像素推断。
- Runtime 已增加不可执行 grounding 门禁，文本合成坐标不会触发动作或资源锁。
- 已实现生产 HTTP Vision Adapter、严格响应协议、图片完整性校验和 pixels-only CLI；图片结果会附加 DOM-like `semantic_tree` 投影。
- 已实现无 KT6 密钥的 CodeAgent CLI Adapter；只有成功的逐图 `read` 事件和严格拓扑 JSON 同时存在时才接受为像素识别。
- 已实现不依赖 Agent 或外部 API 的本地 RapidOCR/OpenCV 单图 Adapter，并验证规则节点、主干分支和设备表下游归属。
- DOM/ARIA 快照新增独立 `ui_tree`；真实页面层级、业务拓扑图和图片 `semantic_tree` 不再混为同一种 DOM。
- 视觉模型识别出的业务 ID 默认仅用于分析，未经生产资产库核验不能执行动作。
